## 1. 项目目标

本项目目标是构建一个基于生成式 AI 的辅助系统，用于术前预测特定医生风格的鼻整形术后效果。

必须同时满足三个硬约束：

1. 风格迁移：复现特定医生的几何审美特征（鼻尖、鼻孔、唇部恢复趋势）。
2. 解剖一致：保持非手术区域与头部姿态的三维透视关系。
3. 身份保持：严格保留患者非手术区域的生物特征。

## 2. 总体技术路线

主干：`Stable Diffusion / LDM`  
风格注入：`LoRA`（医生风格触发词，如 `dr_style`）  
结构约束：`ControlNet-Depth`（深度图约束几何）  
局部编辑：`Inpainting + SAM mask`（仅在鼻部区域重绘）  
可选增强：`IP-Adapter`（纹理与光照语义对齐）

## 3. 理论抓手（实现理由）

1. LDM：在 latent 空间去噪，兼顾生成质量与计算效率。  
2. LoRA：仅训练低秩增量 `ΔW = BA`，降低小样本训练过拟合与底模遗忘风险。  
3. ControlNet：用 depth 条件约束空间结构，减少鼻部生成与原脸拓扑不一致。  
4. Inpainting：仅修改 mask 区域，非 mask 区域保持原图，保障身份一致。  
5. IP-Adapter：补足局部重绘后的纹理连续性，减少“拼接感”。

## 4. 关键实施设定

1. 数据构建  
数据来自单一医生病例，按多视角标准化；建议双尺度裁剪：
- 全脸尺度：学习比例关系
- 鼻部尺度：学习细节纹理

2. 训练目标  
LoRA 训练沿用扩散噪声预测损失：`L = E[||ε - εθ(zt, t, c)||²]`。

3. 推理流程  
步骤为：输入术前图 -> SAM 生成鼻部 mask 并羽化 -> 注入 LoRA/ControlNet 条件 -> 区域去噪 -> 输出预测图。

4. ControlNet 策略  
采用 early stopping：在去噪前段保留较强结构约束，后段逐步释放以允许鼻部风格变化（建议 60%~70% 后衰减至 0）。

## 5. 当前代码与该目标的对应关系

当前仓库已完成“前处理与条件构建”层：

1. 检测对比：`face_detection/batch.py`
2. 局部区域处理：`face_detection/masking.py`
3. SAM 分割：`face_detection/sam_mask.py` + `face_detection/segmentation.py`
4. 统一配置：`face_detection/settings.py`

说明：当前代码重点在检测/掩码/分割流水线，不是最终生成训练脚本；它是后续 LoRA + ControlNet + Inpainting 的数据与条件基础模块。

## 6. 分阶段计划

1. 阶段A（已开展）：稳定多检测器与SAM掩码质量，降低漏检和偏移。  
2. 阶段B（待实施）：构建 LoRA 训练集与触发词体系，完成风格注入。  
3. 阶段C（待实施）：接入 ControlNet-Depth 与 Inpainting 推理链路。  
4. 阶段D（待实施）：引入 IP-Adapter 做纹理一致性增强与主观/客观评估。  

## 7. 当前验收标准

1. 六视角输入可稳定产出可用人脸框、部位框与 SAM mask。  
2. 输出目录保持原始树结构，并生成可追溯统计。  
3. 新机器按 README 安装后可直接复现当前流程。
