# 项目日志（AI 快速接手）

该目录的目标是：让新的 AI/协作者快速理解项目目标、当前状态和下一步动作。

## 推荐阅读顺序

1. `report000.md`：项目总计划与核心目标（先看）
2. `report_001.md`：当前阶段实验结论与实际迭代尝试记录
3. 后续报告：按时间递增补充

## 强约束

1. `report000.md` 为冻结基线文件，禁止修改。  
2. 新阶段只允许新增 `report_00x.md`，不得覆盖 `report000.md`。  
3. 若目标确需调整，必须由项目负责人明确批准后，以“新增报告”方式记录变更。  

## 报告规范（简洁）

每份报告尽量只写：
1. 核心结论
2. 当前推荐配置
3. 已知风险/问题
4. 下一步计划

补充说明：
- `report000.md` 负责定义不变的项目目标与总路线。
- `report_001.md` 及后续报告应尽量记录真实的工程迭代过程，包括做了什么、试了什么、为何保留或淘汰，而不只是最终结论。
